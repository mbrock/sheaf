# Sheaf Agent Notes

- Finish the work end-to-end when practical.
- When the repo is in a coherent state and relevant tests pass, commit and push automatically to the current branch. Do not wait for an extra prompt.
- Run the smallest relevant verification before committing. If verification cannot run, say so clearly and do not pretend the state is green.
- Do not rewrite published history or use destructive git commands unless explicitly asked.
- Keep deployment-specific hosts and secrets out of tracked files.
- The ontology namespace is `https://less.rest/sheaf/`.
- Resource IRIs should come from runtime configuration, not hardcoded repo values.
